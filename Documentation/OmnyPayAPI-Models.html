  • Session

    A Session object representing current transaction session.

    The Session is a temporal container object, which associates a Device and/or a Shopper with a Merchant and Store. It offers a channel over which the Merchant can message the shopper and track their progress from an unauthenticated state, to authenticated state and subsequently through a checkout workflow.

  • BasketState

    An enum representing the state of current basket.

    • New: A state of a new or already existing active basket.
    • CompleteScan: A state for basket when Point of Sale terminal finishes adding line items.
    • InitiatePayment: A state of basket when app starts the payment. See startPayment(withPaymentInstrument:completion:)
    • ReadyForPayment: A state of basket between PoS starts payment process and payment vault responds.
    • CompleteTransaction: A state of basket when transaction is successfully completed.
    • CancelTransaction: A state of basket when transaction is explicitly cancelled.
